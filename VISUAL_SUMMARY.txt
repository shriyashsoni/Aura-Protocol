KEY LEARNINGS
═══════════════════════════════════════════════════════════════════════════════

  Monorepo Management:
  • Root package.json should only define workspace structure
  • Each workspace manages its own dependencies
  • Version conflicts arise from root defining app dependencies

  Dependency Versions:
  • next@15.1.7 uses Webpack (works with custom config)
  • tailwindcss@3.4.14 uses JavaScript (portable, no binary)
  • react@18.3.0 required by wallet adapters
  • Misaligned versions = complete build failure

  CSS Processing:
  • PostCSS pipeline: CSS → tailwindcss → autoprefixer → output
  • Tailwind v4 needs lightningcss (native binary) - not portable
  • Tailwind v3 uses JavaScript - works everywhere
  • CSS processing failure = no styling = white screen

  Build System:
  • Webpack: Custom config possible (wallet adapter support)
  • Turbopack: Faster but config more limited
  • Monorepo: Workspace isolation important
